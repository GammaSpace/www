## Architecture

**Static site for Gamma Space** (gammaspace.ca), a video game creators cooperative. Built with Astro 6, Tailwind CSS v4, deployed on Netlify.

### Content System

Content is **not** using Astro Content Collections. Instead, markdown files in `src/content/` are read manually with `fs.readFileSync()` and parsed with the `marked` library. Each content section is a separate `.md` file.

- `src/content/games/{slug}/` — per-game markdown files (title, author, intro, description, features, impact, about, links, screenshots)
- `src/content/home/` — homepage section content
- `src/content/shared/` — reusable content blocks
- Game images live in `public/content/games/{slug}/`

### Routing

- Game detail pages use `src/pages/games/[slug].astro` with hardcoded `getStaticPaths()` entries
- Adding a new game requires: adding its slug to `getStaticPaths()`, creating `src/content/games/{slug}/` markdown files, and adding assets to `public/content/games/{slug}/`
- Legacy redirects in `public/_redirects` (Netlify format)

### Styling

Tailwind v4 with the Vite plugin (`@tailwindcss/vite`). Global styles in `src/styles/global.css` use `@theme` for custom tokens:
- `--color-gammaYellowBackground: #F2E57C`
- `--color-gammaYellowLight: #FFF6B8`
- `--color-gammaPurpleGray: #E6E6E6`

Custom component classes (`.activity-box`, `.screenshot-box`, `.link-box`) are defined in global.css.

### Components

- `BaseLayout.astro` — wraps all pages; includes Plausible analytics, favicons, Header/Footer
- `ScreenshotImage.astro` — responsive `<picture>` with WebP; expects images generated by the screenshot script
- `ImpactBox.astro` — impact statement with icon
- Markdown content is injected via `<Fragment set:html={...} />`

### Key Conventions

- Node.js >=22.12.0
- TypeScript strict mode
- Font: IBM Plex Sans (Google Fonts)
- No runtime JS frameworks — pure Astro static HTML
